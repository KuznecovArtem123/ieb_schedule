# API расписания ИЭБ

REST API для получения расписания, групп и преподавателей.  
Реализация: `schedule/views.py` (Django REST Framework).

**Базовый URL:** `/api/`

**Формат:** JSON  
**Методы:** только `GET`  
**Авторизация:** не требуется (открытые эндпоинты)

---

## Общие параметры

### `edu` — ступень образования

Используется в эндпоинте `/api/lessons/`.


| Значение | Описание           |
| -------- | ------------------ |
| `spo`    | СПО (по умолчанию) |
| `vo`     | ВО                 |


### `week` — неделя расписания


| Значение | Описание                      |
| -------- | ----------------------------- |
| `this`   | Текущая неделя (по умолчанию) |
| `next`   | Следующая неделя              |


---

## Коды ответов


| Код   | Описание                                             |
| ----- | ---------------------------------------------------- |
| `200` | Успешный запрос                                      |
| `404` | Объект не найден (расписание, группа, преподаватель) |


---

## Эндпоинты

### 1. Уроки по расписанию

Возвращает все уроки для выбранного отделения и недели.

```
GET /api/lessons/
```

#### Query-параметры


| Параметр | Тип    | Обязательный | По умолчанию | Описание          |
| -------- | ------ | ------------ | ------------ | ----------------- |
| `edu`    | string | нет          | `spo`        | `spo` или `vo`    |
| `week`   | string | нет          | `this`       | `this` или `next` |


#### Пример запроса

```http
GET /api/lessons/?edu=spo&week=this
```

#### Пример ответа `200`

```json
[
  {
    "id": 1,
    "subject": "Математика",
    "date": "2026-05-19",
    "weekday": "Понедельник",
    "order": 1,
    "start_time": "08:30:00",
    "end_time": "10:00:00",
    "group_code": "С2125",
    "auditorium": "402",
    "teachers": ["Иванов И.И."]
  }
]
```

#### Поля объекта урока


| Поле           | Тип     | Описание                      |
| -------------- | ------- | ----------------------------- |
| `id`           | integer | ID урока                      |
| `subject`      | string  | Название предмета             |
| `date`         | string  | Дата (ISO `YYYY-MM-DD`)       |
| `weekday`      | string  | День недели (на русском)      |
| `order`        | integer | Номер пары                    |
| `start_time`   | string  | Время начала (`HH:MM:SS`)     |
| `end_time`     | string  | Время окончания (`HH:MM:SS`)  |
| `group_code`   | string  | Код группы                    |
| `auditorium`   | string  | Аудитория (может быть пустой) |
| `teachers`     | array   | Список ФИО преподавателей     |


#### Ошибки

- `404` — расписание с указанными `edu` и `week` не загружено в систему.

---

### 2. Уроки группы

Возвращает уроки конкретной группы за выбранную неделю (по всем отделениям, где есть привязка к `week`).

```
GET /api/lessons/fromGroup/{id}
```

#### Path-параметры


| Параметр | Тип     | Описание  |
| -------- | ------- | --------- |
| `id`     | integer | ID группы |


#### Query-параметры


| Параметр | Тип    | Обязательный | По умолчанию | Описание          |
| -------- | ------ | ------------ | ------------ | ----------------- |
| `week`   | string | нет          | `this`       | `this` или `next` |


#### Пример запроса

```http
GET /api/lessons/fromGroup/3?week=next
```

#### Пример ответа `200`

Формат массива — как в [уроках по расписанию](#1-уроки-по-расписанию).

#### Ошибки

- `404` — группа с указанным `id` не найдена.

---

### 3. Уроки преподавателя

Возвращает все уроки преподавателя (без фильтра по `edu` / `week`).

```
GET /api/lessons/fromTeacher/{id}
```

#### Path-параметры


| Параметр | Тип     | Описание         |
| -------- | ------- | ---------------- |
| `id`     | integer | ID преподавателя |


#### Пример запроса

```http
GET /api/lessons/fromTeacher/5
```

#### Пример ответа `200`

Формат массива — как в [уроках по расписанию](#1-уроки-по-расписанию).

#### Ошибки

- `404` — преподаватель с указанным `id` не найден.

---

### 4. Список преподавателей

```
GET /api/teachers/
```

#### Пример запроса

```http
GET /api/teachers/
```

```json
[
  {
    "id": 1,
    "last_name": "Иванов",
    "first_name": "Иван",
    "patronymic": "Иванович",
    "search_name": "Иванов И.И."
  }

```

#### Поля объекта


| Поле          | Тип     | Описание                  |
| ------------- | ------- | ------------------------- |
| `id`          | integer | ID                        |
| `last_name`   | string  | Фамилия                   |
| `first_name`  | string  | Имя                       |
| `patronymic`  | string  | Отчество                  |
| `search_name` | string  | Строка для поиска в Excel |

| Поле         | Тип     | Описание       |
| ------------ | ------- | -------------- |

## Сводная таблица


| Метод | URL                             | Описание                                        |
| ----- | ------------------------------- | ----------------------------------------------- |
| GET   | `/api/lessons/`                 | Все уроки расписания (`edu`, `week`)            |
| GET   | `/api/lessons/fromGroup/{id}`   | Уроки группы (`week`)                           |
| GET   | `/api/lessons/fromTeacher/{id}` | Уроки преподавателя                             |
| GET   | `/api/groups/`                  | Список **преподавателей** (см. примечание выше) |
| GET   | `/api/teachers/`                | Список **групп** (см. примечание выше)          |

## Примеры использования

### JavaScript (fetch)

```javascript
const response = await fetch('/api/lessons/?edu=spo&week=this');
const lessons = await response.json();
```

### Python (requests)

```python
import requests

r = requests.get('http://localhost:8000/api/lessons/', params={'edu': 'vo', 'week': 'next'})
r.raise_for_status()
lessons = r.json()
```

### cURL

```bash
curl "http://localhost:8000/api/lessons/?edu=spo&week=this"
curl "http://localhost:8000/api/lessons/fromGroup/1?week=this"
curl "http://localhost:8000/api/teachers/"
curl "http://localhost:8000/api/groups/"
```

---

## Зависимости данных

1. Расписание должно быть загружено через админ-панель для нужных `edu` + `week`.
2. В понедельник расписание `next` может быть автоматически перенесено в `this`
